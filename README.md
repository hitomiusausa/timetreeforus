# TimeTree For Us

家族で予定を共有するためのWebカレンダーアプリ。

TimeTreeのように、家族ごとのカレンダーを作り、IDとパスワードでログインして予定を共有する。スマホでの利用を中心に、PCでも見やすい月表示を目指している。

## 現在の構成

- フロントエンド / サーバー: Next.js App Router
- 言語: TypeScript
- DB: Supabase Postgres
- ORM: Prisma
- 公開: GitHub private repository + Vercel Hobby
- デザイン: 独自CSS
- 画像: `public/logo.webp`, `public/roll.webp`
- PWA: Web App Manifest + Service Worker

## データ保存

入力されたデータはSupabase Postgresに保存される。

- ユーザー
- ログインセッション
- 家族カレンダー
- 家族メンバー
- 招待コード
- 予定
- 担当者
- タイトル候補

GitHubにはコードだけを保存し、予定データは保存しない。

## セキュリティ

現在の主な制御:

- 所属しているカレンダーだけ表示
- URLに他のカレンダーIDを入れても、自分の所属カレンダーからのみ選択
- 予定追加 / 編集 / 削除時にカレンダー所属チェック
- 書き出しAPIでもカレンダー所属チェック
- カレンダー名変更 / 削除は管理者のみ
- パスワードはbcryptでハッシュ化
- DB接続情報は`.env`とVercel環境変数で管理

## 起動方法

```bash
npm install
npm run db:push
npm run dev
```

ブラウザで `http://127.0.0.1:3000` を開く。

## 本番公開

GitHub private repository、Vercel Hobby、Supabase Postgresで公開する。

必要な環境変数:

- `DATABASE_URL`: Supabase Postgres Transaction pooler接続文字列
- `DIRECT_URL`: Supabase Postgres Session pooler接続文字列

VercelのBuild Commandは標準の `npm run build`。

GitHubの`main`ブランチにpushすると、Vercelで自動デプロイされる。

## よく使うコマンド

```bash
npm run lint
npm run build
npm run db:migrate
```
